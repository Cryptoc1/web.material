About Web.Material

Welcome

Welcome to Web.Material. Here on this site you can find documentation and other content on Web.Material.

What is Web.Material?

Web.Material is two files that attempt to make Material Design concepts easily implemented into websites. A CSS and a JS file hold functions and classes that make it easy to imlpement responsive features and fluid animations. For more imformation on Material Design, Google has created documentation here.

Who is Web.Material?

Web.Material was made by Samuel Steele <@cryptoc1>. Although, Others can contribute to the project by forking it on GitHub, or submitting suggestions to Sam

Where Can I get Web.Material?

Web.Material is available for download through GitHub.

To get started using Web.material load the CSS and JS files in your webpage. Whether done using JavaScript or the onload attribute, Material.init(config) should be called when the webpage loads. This accepts an obect that defines some global variables.

Creating a Floating Button

One of the simpilest things that can be crated with Web.Material is a floating button. Doing so is as simple as creating a div and giving it a class material-button-floating. In this example an image was put inside the button. By default floating buttons are 50x50 pixels wide. Web.Material assumes that any image put inside the button will be 24x24 pixels. Although, all of these rules can be overidden by custom CSS.

Creating a button with pop-out menu

Begin by creating an element to be a button, in this tutorial it will be an image. Set that elements onclick attribute to Material.menu.toggle({button: 'more-button', menu: 'more-menu', width: 100, height: 200, duration:0.18}). The exact usage for Material.menu.open(config) is defined in the docs. Breaking down this example though, you'll see that we send an object to the function. The first key in the object defines the id of the button element, the next key defines the id of the menu we're trying to access. The last three keys define the dimensions of the menu should be, and how fast it should open.

Overview

The folllowing information is the documentation. It overlooks and explains the available methods and classes that come with Web.Material. Remember that all properties and methods can be overwritten, as long as they comply with standard HTML Presidence Rules. Web.Material is still under development, so please be aware that some functions or classes may not work as expected or planned. If you noticed a reoccurring bug please, please, PLEASE report it here. The colors in these demos are relative to this site's theme.


Initiation
Code:
Material.init({
backgroundColor: '#F5F5F5',
accentColor: '#00E676'
});
Methods:

Material.init(config) -- Initiates Compentents.

config.backgroundColor -- String hex. Defines the background color of the page.

config.accentColor -- String hex. Defines accent color of the page .It's used when coloring Sliders and form like Item.

Styling:

None.

Notes:

When the webpage is loaded the Material.init(config) method must be invoked. This function loads the fonts and sets up some global variables, in particular, the color key defines the accent color for your website. This accent color is used when coloring switches and sliders, they are not coded into the library.

Cards
Example:
Hello World!

This is an example of a Card component.

Code:
<div id="cards-wrapper">
<div class="material-card" title="Hello World" style="margin: 10px; width: 600px;">
<div class="material-card-heading">Hello World!</div>
<div class="material-card-content">
<p>This is an example of a Card component.</p>
</div>
</div>
</div>

<!-- OR -->

<script>
var config = {
parent: 'cards-wrapper',
indent: true,
title: true.
heading: 'Hello World!,
content: '<p>This is an example of a Card component.</p>'
};

Material.card.create(config);
</script>
Methods:

Material.card -- Parent class.

Material.card.create(config) -- Creates a new card.

config.parent -- String id. The element that cards should be appended to.

config.heading -- String. The heading text of the card.

config.content -- String. Content of the card.

config.title -- Boolean. If true the Title Attribute will be the same as config.heading

config.indent -- Boolean. If true, the content of the card will be indented.

Styling:

material-card -- Parent class for a card. Sub-classes should be inside this element.

material-card-heading -- Styles the heading of the card.

material-card-content -- Styles the content of the card.

Notes:

The width of the card should be styled with inline or custom CSS.

Methods:

Material.list -- Parent class.

Material.list.sort(config) -- Method to sort a list. Accepts an object.

config.id -- String id. Id of the material-list element.

config.descending -- Boolean. False by default (meaning ascending is true by default), if true list items will be sorted in descending order.

config.dividers -- Boolen. True by default. If false, dividers will not by added between items when sorted.

Material.list.create(config) -- Method to create a list.

config.id -- String id. Id of the material-list element.

config.heading -- String. Heading of the list item being created.

config.secondary -- String. Secondary text string.

config.dividers -- Boolen. False by default, if true, dividers will be placed in between list items.

Styling:

material-list -- Parent class to lists. Acts as a container for lists.

material-list-item -- An item in a list.

material-list-avatar -- If it is a list of people or items that may be easily identified by images, this is the image that appears to the left of the list's text.

material-list-content -- Parent class to a list's text (heading, and secondary text).

material-list-heading -- Heading for the list item.

material-list-secondary -- Secondary text of a list item.

material-list-icon -- An action icon within a list. Icon is a material-icon

Notes:

Like many other components, lists have a heirarchy structure that defines how items should appear to the user. The Material.list.sort(config) should be called on page load. List items are sorted by the value of the material-list-item's name attribute. If the name atrribute contains a String, the list items will be sorted alphabetically. The sort method supports numbers.
